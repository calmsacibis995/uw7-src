SCO Sound Design Specification

Shawn McMurdo
1996 Nov 06

Tasks

  • Create a new empty current soundfile.
  • Open an existing soundfile as the current soundfile.
  • Save the current soundfile with a different name.
  • Play the current soundfile from the beginning.
  • Begin automatically playing a soundfile on startup.
  • Stop the current soundfile from playing or recording.
  • Pause the current soundfile from playing or recording.
  • Resume playing or recordin the current soundfile from being paused.
  • Record over the current soundfile.
  • Change the audio device being used to play or record soundfiles.
  • Change the default parameters used in creating a new soundfile.
    • File Format
    • Date Format
    • Sample Rate
    • Channels

Commandline 


usage: scosound [options] [filename]

options include:
-audiodevice (device)		audio device
-audio (device)			audio device
-device (device)		audio device
-fileformat (fileformat)	file format (default: au)
-file (fileformat)		file format (default: au)
-dataformat (dataformat)	data format (default: ulaw)
-data (dataformat)		data format (default: ulaw)
-comment (string)		comment (default: "")
-samplerate (rate)		sampling rate (default: 8000)
-rate (rate)			sampling rate (default: 8000)
-gain (percent)			input gain in percent (default: 75)
-volume (percent)		output volume in percent (default: 75)
-channels (channels)		number of channels (default: 1)
-mono				same as -channels 1
-stereo				same as -channels 2
-autoplay			automatically initiate playing of the file

(device) is a device node name such as /dev/dsp0 or /dev/audio.

(fileformat) is one of the following:
aiff	AIFF
au	Sun/NeXT mulaw
snd	raw
voc	Creative Labs VOC
wav	Microsoft WAVE

(dataformat) is one of the following:
pcm8			common unsigned 8 bit linear
u8			unsigned 8 bit linear
s8			signed 8 bit linear
pcm16			common signed little endian 16 bit linear
s16l			signed little endian 16 bit linear
s16b			signed big endian 16 bit linear
u16l			unsigned little endian 16 bit linear
u16b			unsigned big endian 16 bit linear
mulaw			common mu law 8 bit logarithmic
ulaw			common mu law 8 bit logarithmic
alaw			a law 8 bit logarithmic
ima_adpcm		IMA adaptive delta
mpeg			MPEG

Note that the duplicate names for some of the options and format types
are for clarity and compatibility with other audio clients.

(string) can be any string to be embedded in the file's comment field,
if supported by the file format.

(rate) is one of the following:
8000
11025
16000
22050
32000
44100
48000

(percent) is an integer value from 0 to 100.

(channels) is the number of channels where 1 is mono and 2 is stereo.
